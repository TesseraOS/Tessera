Four reference sets, one property: **everything here is generated from the running
system and drift-checked in CI.** The REST pages render from the server's own
OpenAPI document; the MCP catalog is what the real server answered to `tools/list`;
the CLI pages render from the same table `tessera help` prints; the configuration
tables parse the repository's canonical `.env.example`. If code changes and the docs
don't, the build goes red — these pages cannot quietly lie.
